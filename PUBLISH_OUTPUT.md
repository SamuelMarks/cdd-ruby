# Publishing the Output Clients

This document focuses on the client-library that `cdd-ruby` generates from a `spec.json` OpenAPI file, and keeping that client-library up-to-date with the server.

## Continuous Synchronization

To ensure the client SDK always matches the source truth, you can set up a GitHub Action cron job or web hook in the client repository.

### GitHub Actions Workflow (Client Side)

```yaml
name: Sync Client SDK from Server OpenAPI

on:
  schedule:
    - cron: '0 0 * * *' # Daily sync
  workflow_dispatch:

jobs:
  update-sdk:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3

      - name: Setup Ruby
        uses: ruby/setup-ruby@v1
        with:
          ruby-version: '3.4'

      - name: Install cdd-ruby
        run: gem install cdd-ruby

      - name: Download Server OpenAPI Spec
        run: curl -sL https://my-server.com/openapi.json -o spec.json

      - name: Regenerate SDK
        run: cdd-ruby from_openapi to_sdk -i spec.json -o ./lib

      - name: Commit and Push changes
        run: |
          git config user.name "github-actions[bot]"
          git config user.email "41898282+github-actions[bot]@users.noreply.github.com"
          git add ./lib
          git commit -m "Auto-update client SDK" || echo "No changes to commit"
          git push
```

## Publishing the Generated Client

Once generated by `cdd-ruby`, the client library is just standard Ruby code.
You will need to write a `.gemspec` for it manually or modify the one that `cdd-ruby` could potentially scaffold.
You can publish it using the standard `gem push` command to `rubygems.org`.
